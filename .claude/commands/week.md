# Weekly Review (Sunday)

Read docs/BUILD_LOG.md and CLAUDE.md silently.

## Output these sections:

### Completed This Week
List all tasks from BUILD_LOG.md completed in the last 7 days. Format:
- [Task ID] — [Description] ([Date])

### Blockers
List any current blockers from CLAUDE.md or Known Issues in BUILD_LOG.md.
If none: "None."

### Priority for Next Week
Identify the single highest-priority item for the coming week based on:
- Current sprint progress
- Known issues severity
- Any blockers that need resolving

### CLAUDE.md Update
Draft a 4-line update for the build status section of CLAUDE.md:
```
Current sprint: Sprint X
Current task: TY
Last completed: TZ [description]
Blocking issues: [issues or NONE]
```

### LinkedIn Post Draft
Write a short LinkedIn post about this week's progress. Rules:
- Direct and honest tone
- No hype, no buzzwords
- No "journey", "excited to share", or "thrilled"
- Focus on what was built and what was learned
- Keep it under 150 words
- End with what's next, not a call to action

Ask: "Apply the CLAUDE.md update? (yes/no)"

If yes, update CLAUDE.md with the drafted 4-line update.
